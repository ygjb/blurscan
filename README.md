# blurscan

> This project was created as a demo of how to use Claude to quickly and easily set up a loop to create a complex tool.  It is almost completely generated by Claude, and is using a Claude managed CI/loop to handle all implementation details with a human in the loop for design decisions.  You can see the related [blog post on https://parentsguidetoai.ca](https://blog.parentsguidetoai.ca/building-a-tool-with-claude-blurscan.html)

A Python CLI that scans a photo collection, scores each image's sharpness, and flags
blurry images — while sparing intentional shallow-depth-of-field shots. Flagged images
can be reported, quarantined, tagged (XMP via exiftool), or triaged in a local web UI.

## Install

```bash
pip install -e .            # core (numpy, opencv, pillow, pillow-heif, rawpy, flask)
pip install -e ".[ml]"      # optional: adds the learned `ml` detector (torch + torchvision)
pip install -e ".[dev]"     # ruff + mypy + pytest
```

The `--tag` action needs **exiftool** on your PATH (`apt install libimage-exiftool-perl`,
`brew install exiftool`). Everything else is pure pip.

## Quickstart

```bash
# Report only (default) — prints a ranked summary, writes CSV + HTML:
blurscan ~/Photos --report report

# Copy flagged images aside to review by hand (safe; originals untouched):
blurscan ~/Photos --copy ~/Photos/_blurry

# Tag flagged images in your library so Lightroom/digiKam can filter them:
blurscan ~/Photos --tag

# Triage interactively in the browser (per-tile heatmaps, keyboard decisions):
blurscan ~/Photos --review
```

By default blurscan is **report-only** — it changes nothing unless you pass an action,
and every destructive action honors `--dry-run`.

## Detection methods (`--method`)

Each image gets a sharpness score (higher = sharper); the method is pluggable because no
single heuristic fits every collection. ROC-AUC below is on the bundled CC test corpus.

| `--method`   | How it works | AUC | Best for |
|--------------|--------------|-----|----------|
| `laplacian` (default) | Tiled **max** variance-of-Laplacian (sharpest region) | 0.93 | General / out-of-focus blur |
| `motion`     | Orientation-aware: weakest-direction gradient energy | 0.87 | Residual subject **motion / panning** blur |
| `ml`         | Frozen MobileNetV3 features + logistic head (needs `[ml]`) | 0.95 | Content-specific blur, best overall |

## Actions (combine freely)

| Flag | Effect |
|------|--------|
| `--report PATH` | Writes `PATH.csv` (all scores) + a self-contained `PATH.html` gallery, blurriest-first |
| `--copy DIR` / `--move DIR` | Quarantine flagged images (copy is reversible); collision-safe names |
| `--tag` | XMP keyword `blurscan:<class>` + rating via exiftool; RAW → XMP sidecar (`--raw-inplace` to override) |
| `--review` | Local web UI on `127.0.0.1` (`--port`, `--no-open`) |
| `--include-borderline` | Act on borderline images too (default: blurry only) |
| `--dry-run` | Show what every action would do, change nothing |

## Tuning the threshold

Thresholds **don't transfer across methods or collections** — each method has its own
scale (laplacian/ml ~100/50; motion in the thousands). To calibrate to *your* library:

1. **Look at the scores first.** `blurscan DIR --json` (or the CSV report) prints every
   image's `score_max_tile`. Sort it and eyeball where blurry gives way to sharp.
2. **Set an absolute floor** with `--threshold N` once you've found the knee.
3. **Or go relative** with `--adaptive [PCT]` — flags the bottom `PCT`% of *this run's*
   distribution as borderline (default 10%). This is the most robust mode when a method
   ranks well but no single cut separates cleanly. It never overrides a hard `--threshold`
   call to blurry.
4. Re-run freely: results are cached (`--no-cache` to force), so iterating on the
   threshold is instant (classification is recomputed each run).

## Performance

Scoring is parallelized across CPU cores (`--jobs N`) and cached on disk under
`<scan>/.blurscan_cache/` keyed on path+mtime+size, so reruns only re-score changed files.
RAW decodes the embedded preview by default (fast); `--raw-full` demosaics the sensor data.
